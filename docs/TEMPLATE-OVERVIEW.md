## What's ready vs stubbed

**Ready to run:**

- Nuxt 4 / Tailwind 4 web app — `docker compose up web` works after `.env` is set
- Puppeteer sidecar — boots and serves `/health` + `/render` after first `npm install` inside container

**Needs one-time bootstrap:**

- `apps/api/` is **empty** of Laravel code by design. Running `bash apps/api/bootstrap.sh` installs Laravel 11, applies overrides, installs `spatie/laravel-data`, generates `APP_KEY`. Removes itself after. **The template doesn't ship 100+ Laravel framework files** — bootstrap script handles that on first clone.

## Decisions baked in

- No `@nuxtjs/tailwindcss` (Tailwind 4 via `@tailwindcss/vite`)
- No `tailwind.config.ts` (CSS-first `@theme` block in `main.css`)
- No Sanctum on the API; token-only auth via `AdminTokenMiddleware`
- No Redis in the template; add only if a specific tool actually needs it
- No Browsershot in the API; Chromium runs **only** in the puppeteer sidecar
- All host ports bound to `127.0.0.1`
- Container names parameterized via `${PROJECT_SLUG}` (e.g. `my-tool-web-dev`, `my-tool-api-dev`)
- Volumes are named: `web_node_modules`, `api_vendor`, `puppeteer_node_modules`, `puppeteer_chromium_cache` — persist across restarts
- Services join external Docker network `axelnova-shared` (declared in `axelnova-infra/docker-compose.yml`)
- TypeScript strict everywhere; PSR-12 for PHP
- Default ports for the "first tool": web `3010`, api `8010` (claim from [axelnova-infra/docs/port-allocation.md](../../axelnova-infra/docs/port-allocation.md) — Tools range `3010–3049 / 8010–8049`)

## Known limitations / things to verify on first real use

1. **Puppeteer Chromium download** in dev: first `docker compose up puppeteer` will install puppeteer which downloads Chromium (~170MB) into the named volume. Slow first time, fast after.
2. **`apps/api/.gitignore`** isn't shipped — Laravel's own `.gitignore` is generated by `bootstrap.sh`. Root `.gitignore` already covers `vendor/`, `storage/*`, `bootstrap/cache/*`.
3. **Deploy workflow assumes the repo is already cloned on the VPS** at `~/<repo-name>` with a `.env` present. First deploy is manual: SSH in, `git clone`, set up `.env`, run bootstrap, then GitHub Actions takes over.
4. **`build-check.yml` runs `npm ci`** which needs `package-lock.json` committed. The template doesn't ship lockfiles — first push from a new tool repo should commit them.
